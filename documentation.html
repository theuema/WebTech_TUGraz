Edu Image Tool Documentation

Educational Image Tool - Documentation

With this application we try to explain non-specialists the concept of digital images in the RGB color space. For each supported operation we provide an information box describing the technical background of the performed operation.

Note: To allow WebWorkers to work properly, you need to run the application on a local web server. For instance, change to the application directory and run 'python3 -m http.server'. The application is now available at localhost:8000/eduimage.html

HTML5 Technologies

Semantic Markup

In order to structure our application we used HTML5 semantic markup. More specifically, we used the 'nav' tag for our toolbox and the 'footer' tag for some additional information.

WebForm Components

To allow the user to interact with the image operations we implemented WebForm components. For instance, it is possible to change the alpha value of the uploaded image via a HTML5 slider component.

FileSystem API

The FileSystem API enables the user to upload an image from his/her local harddrive to our application. Furthermore, after playing around with our image operations, we provide the user with a 'save' button to store the modified image persistently.

Drag & Drop

In addition to uploading an image via the FileSystem API we provide the user the well-known Drag & Drop functionality. Simply drag an image from your local filebrowser and drop it inside the marked spot of our application.

Canvas Element

The heart of our application is editing an image including an explanation of the operations. To draw the image we use the HTML5 canvas element.
We support the following operations:

  • Splitting the image into its RGB channels
  • Convert the image into greyscale
  • Compute the negative of the image
  • Convert the image into Black & White
  • Manipulate the alpha value of the image
  • Move the different channel images via drag and drop

WebWorkers

Image processing is a perfect example for 'single instruction multiple data'. We implement WebWorkers to handle the image operation in a multi-threading manner. The image data array is split into equal parts and each WebWorkers processes the data in parallel. After we receive all the results from the WebWorkers we can redraw the canvas and show the final result

Architecture

In order to achieve the separation of content, presentation and behaviour we use the MVC concept and structured our source code accordingly.

View

The HTML and CSS files include the implemenation of the 'view' layer.

Model

We use Javascript to interact with the FileSystem API.

Controller

Furthermore, Javascript helps us to combine the image operations on our model and finally present it to our view
